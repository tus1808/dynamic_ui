## Project Overview

`dynamic_ui` is a GTK3/C application for Raspberry Pi 3 Model B (Raspberry Pi OS / Linux) that displays real-time UART sensor data on a configurable canvas. An editor mode (password-protected) lets users drag, resize, and reposition value items at runtime, with layout persisted to JSON.

## Build Commands

```bash
# Install dependencies (on target device)
sudo apt install -y build-essential meson ninja-build pkg-config libgtk-3-dev libjson-glib-dev

# First-time setup
meson setup build

# Reconfigure (after meson.build changes)
meson build --reconfigure

# Compile
meson compile -C build

# Run
cd build && ./app
```

There are no tests and no linter configured. The VSCode tasks in `learn-c/orangepi-uart/.vscode/tasks.json` demonstrate SCP + SSH workflows for deploying and running on the Raspberry Pi at `192.168.1.102`.

## Architecture

The application is organized into six modules under `src/`:

```
src/
  main.c              — entry point, creates GtkApplication + AppController
  app/                — lifecycle and shared state
  mode/               — READ / EDITOR mode switching
  ui/                 — GTK3 widgets (window, canvas, overlay, toolbar, items)
  uart/               — serial port I/O and frame parsing
  config/             — JSON config + layout loading/saving
  input/              — keyboard hotkeys and editor password auth
  common/             — shared types (AppConfig, LayoutItem, UartFrame) and constants
```

### Data flow

1. `main.c` creates `AppController` which owns `AppState`.
2. `AppState` holds a pointer to the GTK window, canvas, overlay, loaded config, and current layout.
3. `uart/port.c` auto-detects the serial device and starts a read loop. Each 136-byte frame (8-byte header + 128-byte body) is parsed by `uart/parser.c` and dispatched via `uart/data_dispatcher.c` to update `LayoutItem` values in state.
4. `mode/manager.c` arbitrates between `APP_MODE_READ` and `APP_MODE_EDITOR`. Switching to EDITOR requires SHA-256 password verification in `input/auth_manager.c` (hash stored in `config/app.json`).
5. In READ mode, `ui/read_mode.c` renders value items on the canvas. In EDITOR mode, `ui/editor_mode.c` enables drag/resize via `ui/resize_handle.c`, and `ui/editor_toolbar.c` exposes save/revert actions that flush layout back to `config/layout.json` through `config/layout_store.c`.

### Key types (`src/common/types.h`)

- `AppConfig` — paths (background, CSS, layout file), window title, hashed editor password
- `LayoutItem` — `_id`, `x`, `y`, `width`, `height`, `value` (hex string)
- `UartFrame` — fixed 136-byte structure
- `AppMode` — `APP_MODE_READ` / `APP_MODE_EDITOR`

### Configuration files

- `config/app.json` — application-level settings including `editor_password` (SHA-256 hex)
- `config/layout.json` — array of `LayoutItem` objects persisted on editor save
- `assets/app.css` — GTK3 stylesheet applied at startup

## Development Notes

- The project is developed on Windows but **targets Linux on Raspberry Pi 3 Model B**. The VSCode C/C++ config (`dynamic_ui/.vscode/c_cpp_properties.json`) points to a Linux gcc path — expect IntelliSense to be approximate on Windows.
- `build/compile_commands.json` is generated by Meson and used by clangd / C/C++ extension for accurate indexing.
- The `learn-c/orangepi-uart/` sibling directory is a standalone learning prototype and is unrelated to the main build.
